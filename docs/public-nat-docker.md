# Public NAT Docker deployment

This deployment publishes GoClaw at `https://io.supermeo.com:88` and keeps every
internal service off the public network. Only the reverse proxy maps a host port.

## Files

- `docker-compose.public.yml`: full public stack for Docker Desktop.
- `deploy/nginx/goclaw-public.conf`: TLS reverse proxy for GoClaw HTTP, UI, and WebSocket traffic.
- `deploy/certs/`: put `io.supermeo.com.crt` and `io.supermeo.com.key` here.

## Prepare

```bash
./prepare-env.sh
```

Edit `.env` and set:

```dotenv
POSTGRES_PASSWORD=<strong-random-password>
POSTGRES_PASSWORD_DSN=<url-encoded-postgres-password>
GOCLAW_ALLOWED_ORIGINS=https://io.supermeo.com:88
DOCKER_GID=0
PUBLIC_HTTPS_PORT=88
```

On macOS, TCP port `88` may already be used by the system Kerberos KDC service.
If so, set `PUBLIC_HTTPS_PORT=8443` and configure the router as
`WAN:88 -> MAC_LAN_IP:8443`. The public browser URL remains
`https://io.supermeo.com:88`, so keep `GOCLAW_ALLOWED_ORIGINS` unchanged.

`GOCLAW_GATEWAY_TOKEN` and `GOCLAW_ENCRYPTION_KEY` are generated by
`prepare-env.sh` when missing. Keep the gateway token private; it is the initial
admin token for the dashboard/API.

Place TLS files:

```text
deploy/certs/io.supermeo.com.crt
deploy/certs/io.supermeo.com.key
```

Because the public URL uses HTTPS on port `88`, automatic HTTP-01/TLS-ALPN
certificate issuance normally will not work unless ports `80` or `443` are also
available. Use an existing certificate or issue one with a DNS-01 flow.

## Build and start

```bash
make public-up
```

## NAT/router

Configure DNS and NAT:

- `io.supermeo.com` points to the WAN IP.
- The Mac has a fixed LAN IP.
- Forward TCP `WAN:88` to `MAC_LAN_IP:${PUBLIC_HTTPS_PORT}`.
- Do not forward `18790`, `5432`, `6379`, `9222`, `16686`, `4317`, or `4318`.

## Verify

```bash
docker compose -f docker-compose.public.yml ps
curl -k https://localhost:88/health
curl -k https://io.supermeo.com:88/health
make public-logs
```

Open `https://io.supermeo.com:88` and log in with the gateway token from `.env`.

## Notes

The stack includes Postgres, Redis, Chrome browser automation, Jaeger telemetry,
and Docker sandbox support. Jaeger is intentionally internal-only in this public
profile. If you need its UI, access it through a temporary localhost-only debug
override rather than forwarding it through the router.

Claude CLI is opt-in. To include it, set `ENABLE_CLAUDE_CLI=true` before build
and add the read-only `~/.claude` credential mount from
`docker-compose.claude-cli.yml`.
